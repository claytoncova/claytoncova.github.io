Introdução às Redes de Computadores

O que são Redes de Computadores?

Uma rede de computadores é um conjunto de dispositivos interconectados que podem comunicar-se e compartilhar recursos.

Benefícios das Redes:

  • Compartilhamento de recursos - impressoras, arquivos, conexão à internet
  • Comunicação - email, mensagens instantâneas, videoconferência
  • Centralização - backup, segurança, administração
  • Economia - redução de custos com hardware
  • Flexibilidade - acesso remoto e mobilidade

Conceitos Fundamentais:

Modelo OSI (Open Systems Interconnection)

Desenvolvido pela ISO em 1984, o modelo OSI divide a comunicação em rede em 7 camadas distintas.

As 7 Camadas do Modelo OSI:

  • 7. Aplicação - Interface com o usuário
  • 6. Apresentação - Criptografia, compressão
  • 5. Sessão - Controle de sessões
  • 4. Transporte - Confiabilidade fim-a-fim
  • 3. Rede - Roteamento
  • 2. Enlace - Controle de acesso ao meio
  • 1. Física - Transmissão de bits

Modelo TCP/IP

Modelo prático usado na Internet, desenvolvido pelo DoD (Departamento de Defesa dos EUA). Possui 4 camadas.

As 4 Camadas do Modelo TCP/IP:

  • 4. Aplicação - Aplicações de rede (HTTP, FTP, SMTP)
  • 3. Transporte - TCP, UDP
  • 2. Internet - IP, ICMP, ARP
  • 1. Acesso à Rede - Ethernet, WiFi, Bluetooth

Observação: O modelo TCP/IP é mais prático e amplamente utilizado, enquanto o OSI é mais teórico e educacional.

Comparação: OSI vs TCP/IP

Modelo OSI

  • 7 camadas
  • Modelo teórico
  • Educacional
  • Mais detalhado
  • Padrão ISO

Modelo TCP/IP

  • 4 camadas
  • Modelo prático
  • Base da Internet
  • Mais simples
  • Padrão de facto

Mapeamento: Aplicação/Apresentação/Sessão (OSI) → Aplicação (TCP/IP)

Camada 1: Física

Função:

Responsável pela transmissão de bits através do meio físico.

Características:

  • Define características elétricas, mecânicas e funcionais
  • Especifica voltagens, correntes e frequências
  • Define conectores e cabos
  • Não interpreta dados - apenas transmite bits

Exemplos de Tecnologias:

  • Cabo de par trançado - Cat5e, Cat6
  • Fibra óptica - Monomodo, multimodo
  • Wireless - WiFi, Bluetooth
  • Cabo coaxial - TV a cabo

Camada Física: Bluetooth e WiFi

Bluetooth

  • Frequência: 2.4 GHz (ISM)
  • Alcance: 1-100 metros
  • Técnica: FHSS (Frequency Hopping)
  • Versões: 1.0 até 5.3
  • Aplicações: Periféricos, áudio

WiFi (IEEE 802.11)

  • Frequências: 2.4/5/6 GHz
  • Alcance: 30-100 metros
  • Técnica: DSSS, OFDM
  • Padrões: a/b/g/n/ac/ax
  • Aplicações: LANs sem fio

Camada Física: Comandos Práticos

Verificação de Interface de Rede:

# Windows ipconfig /all # Linux ip link show ifconfig -a lshw -class network

Status Bluetooth e WiFi:

# Windows netsh wlan show profiles netsh bluetooth show status # Linux bluetoothctl list iwconfig nmcli dev wifi

Informações de Hardware:

# Linux lspci | grep -i network lsusb | grep -i bluetooth dmesg | grep -i wifi

Camada 2: Enlace de Dados

Função:

Fornece comunicação confiável entre nós adjacentes através do meio físico.

Responsabilidades:

  • Framing: Delimita e organiza os dados em quadros
  • Endereçamento: Endereços MAC (48 bits)
  • Controle de Acesso: CSMA/CD, CSMA/CA
  • Detecção de Erros: CRC, checksum
  • Controle de Fluxo: Evita sobrecarga do receptor

Protocolos Principais:

Ethernet, WiFi, PPP, Frame Relay, ATM

Camada de Enlace: Exemplos Práticos

Ethernet (IEEE 802.3)

  • Endereços MAC únicos globalmente
  • CSMA/CD para detecção de colisões
  • Velocidades: 10 Mbps a 400 Gbps
  • Quadros de 64 a 1518 bytes

WiFi (IEEE 802.11)

  • CSMA/CA para evitar colisões
  • WEP, WPA, WPA2, WPA3 (segurança)
  • SSID para identificação da rede
  • Handshake de 4 vias para autenticação

Camada de Enlace: Comandos Práticos

Verificação de Endereços MAC:

# Windows getmac /fo table ipconfig /all # Linux ip link show cat /sys/class/net/*/address

Análise de Tráfego Ethernet:

# Linux tcpdump -i eth0 -e wireshark (interface gráfica) # Windows netsh trace start capture=yes

Configuração WiFi:

# Linux nmcli dev wifi list nmcli dev wifi connect "SSID" password "senha" # Windows netsh wlan connect name="SSID"

Camada 3: Rede

Função:

Responsável pelo roteamento de pacotes entre redes diferentes.

Características Principais:

  • Endereçamento lógico: Endereços IP
  • Roteamento: Determina o melhor caminho
  • Fragmentação: Divide pacotes grandes
  • Controle de Congestionamento: Gerencia tráfego

Protocolos Principais:

  • IP (IPv4/IPv6): Protocolo principal
  • ICMP: Mensagens de controle
  • ARP: Resolução de endereços
  • OSPF, BGP, RIP: Protocolos de roteamento

Camada de Rede: Exemplos Práticos

IPv4

  • 32 bits (4 octetos)
  • Exemplo: 192.168.1.1
  • Classes A, B, C
  • Máscara de sub-rede
  • NAT para conservar IPs

ICMP

  • Ping (echo request/reply)
  • Traceroute
  • Mensagens de erro
  • MTU discovery

IPv6

  • 128 bits (8 grupos)
  • Exemplo: 2001:db8::1
  • Autoconfiguração
  • Sem NAT necessário
  • IPSec integrado

ARP

  • IP → MAC
  • Cache ARP
  • Broadcast requests
  • Unicast replies

Camada de Rede: Comandos Práticos

Configuração e Verificação de IP:

# Windows ipconfig ipconfig /release ipconfig /renew # Linux ip addr show ip route show dhclient eth0

Testes de Conectividade:

# Ambos os sistemas ping 8.8.8.8 ping -6 2001:4860:4860::8888 tracert google.com (Windows) traceroute google.com (Linux)

Cache ARP:

# Windows arp -a # Linux ip neigh show arp -a

Camada 4: Transporte

Função:

Fornece comunicação fim-a-fim confiável entre aplicações.

Responsabilidades:

  • Segmentação: Divide dados em segmentos
  • Controle de Fluxo: Janela deslizante
  • Controle de Erro: Detecção e correção
  • Multiplexação: Múltiplas conexões por processo
  • Controle de Congestionamento: Evita sobrecarga

Protocolos Principais:

TCP (confiável), UDP (rápido), SCTP, DCCP

Camada de Transporte: TCP vs UDP

TCP (Transmission Control Protocol)

  • Confiável: Garante entrega
  • Orientado à conexão: 3-way handshake
  • Controle de fluxo: Evita sobrecarga
  • Ordenação: Sequência de dados
  • Uso: HTTP, FTP, SSH, Email

UDP (User Datagram Protocol)

  • Não confiável: Best effort
  • Sem conexão: Datagramas independentes
  • Sem controle de fluxo: Rápido
  • Sem ordenação: Pode chegar fora de ordem
  • Uso: DNS, DHCP, streaming, jogos

Portas: HTTP (80), HTTPS (443), SSH (22), FTP (21), DNS (53)

Camada de Transporte: Comandos Práticos

Visualização de Conexões:

# Windows netstat -an netstat -b (mostra processos) # Linux ss -tuln netstat -tuln lsof -i

Teste de Portas:

# Windows telnet google.com 80 Test-NetConnection google.com -Port 443 (PowerShell) # Linux telnet google.com 80 nc -zv google.com 80 nmap -p 80,443 google.com

Monitoramento de Tráfego:

# Linux tcpdump -i any port 80 ss -i (estatísticas de conexão) # Windows netsh int tcp show global

Camada 5: Sessão

Função:

Gerencia sessões de comunicação entre aplicações.

Responsabilidades:

  • Estabelecimento de sessão: Início da comunicação
  • Manutenção: Controle durante a sessão
  • Sincronização: Pontos de controle
  • Terminação: Encerramento controlado
  • Recuperação: Retomada após falhas

Exemplos de Protocolos/Tecnologias:

  • SQL sessions: Conexões de banco de dados
  • RPC: Remote Procedure Call
  • NetBIOS: Network Basic Input/Output System
  • SSH sessions: Sessões seguras

Camada 6: Apresentação

Função:

Responsável pela formatação, criptografia e compressão dos dados.

Responsabilidades:

  • Codificação: ASCII, Unicode, EBCDIC
  • Criptografia: SSL/TLS, AES, RSA
  • Compressão: ZIP, GZIP, JPEG
  • Conversão de formato: Entre diferentes sistemas

Exemplos Práticos:

  • HTTPS: HTTP com SSL/TLS
  • MIME: Multipurpose Internet Mail Extensions
  • PNG, JPEG: Formatos de imagem
  • MPEG: Compressão de vídeo

Camada 7: Aplicação

Função:

Interface entre aplicações de rede e usuários.

Características:

  • Fornece serviços de rede para aplicações
  • Interface direta com o usuário
  • Implementa protocolos específicos de aplicação
  • Gerencia comunicação entre processos

Categorias de Aplicações:

  • Web: HTTP, HTTPS
  • Email: SMTP, POP3, IMAP
  • Arquivos: FTP, TFTP, SFTP
  • Terminal remoto: SSH, Telnet
  • Nomes: DNS

Camada de Aplicação: Protocolos Principais

HTTP/HTTPS (Web)

  • Porta 80 (HTTP), 443 (HTTPS)
  • Métodos: GET, POST, PUT, DELETE
  • Stateless por natureza
  • HTTP/2, HTTP/3 (QUIC)

FTP (File Transfer)

  • Porta 21 (controle), 20 (dados)
  • Modo ativo vs passivo
  • SFTP (seguro via SSH)
  • FTPS (seguro via SSL/TLS)

DNS (Domain Name System)

  • Porta 53 (UDP/TCP)
  • Resolução hierárquica
  • Tipos: A, AAAA, MX, CNAME
  • Cache e TTL

Email (SMTP/POP3/IMAP)

  • SMTP: porta 25, 587, 465
  • POP3: porta 110, 995
  • IMAP: porta 143, 993
  • MIME para anexos

Camada de Aplicação: Comandos Práticos

Testes Web e DNS:

# Ambos os sistemas nslookup google.com dig google.com (Linux) # Teste HTTP curl -I http://google.com wget --spider http://google.com (Linux)

Transferência de Arquivos:

# FTP ftp ftp.example.com # SSH/SCP ssh user@server scp arquivo.txt user@server:/destino/

Monitoramento de Serviços:

# Windows sc query Get-Service (PowerShell) # Linux systemctl status ps aux | grep -i ssh netstat -tlnp

Resumo e Conclusões

Pontos-chave Aprendidos:

  • Modelo OSI: Framework teórico de 7 camadas
  • TCP/IP: Modelo prático de 4 camadas usado na Internet
  • Cada camada tem responsabilidades específicas
  • Protocolos implementam as funcionalidades de cada camada
  • Ferramentas de linha de comando permitem diagnóstico e configuração
